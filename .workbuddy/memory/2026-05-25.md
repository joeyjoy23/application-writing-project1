## Stage API 90秒超时 + 自动重试 + 跳过机制

- 在 `app.py` 新增 `_STAGE_TIMEOUT_SECONDS = 90` 常量
- 重写 `_start_api_thread` 的 worker：每次 API 调用放在子线程中，用 `thread.join(timeout=90)` 守卫；超时则自动重试一次（共2次机会）；仍超时则设置 `("timeout", msg)` 错误

- `advance_run_job` 新增 `("timeout", ...)` 错误处理分支：跳过当前 Stage、保留已有结果、继续下一个 Stage；若已是最后一个则结束流程
