# 用户使用说明

面向教师的日常操作说明；安装与部署见 [README.md](../README.md)。

## 1. 新建分析

1. 侧边栏选择 **新建**，配置 API Key 与模型。
2. 粘贴题目，可选 **学生水平**（仅影响 Stage 4）。
3. 点击 **完整流程** 或单独 **Stage 1–4**。
4. 生成内容在下方逐块显示；侧边栏 **已生成内容索引** 可点击跳转到对应 Stage。

全流程时 **Stage 2 与 Stage 3 自动并行**，缩短等待时间。

## 2. 四阶段产出

| 阶段 | 产出 |
|------|------|
| Stage 1 | 审题总结 + 结构化 JSON（后续阶段的数据契约） |
| Stage 2 | PEEL 策略卡、三版范文（**105–125 英文词**）、对比分析 |
| Stage 3 | 功能句型包、话题词汇（必备 / 进阶 / 亮点） |
| Stage 4 | 学情适配教学指南、易错预警、课后练习 |

Prompt 模板在 `prompts/`，修改后重跑对应 Stage 即可生效（`prompt_rev` 变化会使 LLM 缓存失效）。

## 3. 历史记录（自动保存）

**每完成一个 Stage**（至少已有 Stage 1）会自动写入 **历史**，无需手动保存。

| 规则 | 说明 |
|------|------|
| 同题 + 同模型 | **合并为一条**，后续 Stage 会更新同一条记录 |
| 同题 + 换模型 | **另存一条**新记录 |
| 内容未变 | 跳过重复写入（指纹去重） |
| 中断 / 停止 | 已完成阶段仍会保存，可在历史中载入续跑 |

首次保存成功时通常会弹出 **「已保存到历史 #编号」**；运行日志中可见「已同步历史（已完成 x/4…）」。

本地模式历史在 `history.db`；云端部署见 [NEON_SETUP.md](NEON_SETUP.md)。

## 4. 「清空当前结果」与「删除历史」

这是两件**完全不同**的事：

| 操作 | 位置 | 作用 |
|------|------|------|
| **清空当前结果** | 新建页「清空结果」或侧边栏「清空当前结果」 | 只清除**当前页面**的四阶段输出与断点；**不删除**历史库中的记录（两处入口均需二次确认） |
| **删除** | 历史列表每行的「删除」 | **永久删除**该条历史备课包 |

清空后题目、学生水平、侧边栏模型设置仍保留；历史里之前的版本仍可查看、导出或 **载入** 到新建页。

## 5. 更换模型

| 场景 | 行为 |
|------|------|
| 已有结果，侧边栏换了模型 | 页面提示来源模型与当前不一致 |
| 点击 **完整流程** | 清空当前结果，用**新模型**重跑全部四阶段 |
| 点击 **Stage 1** | 清空后只重跑 Stage 1 |
| 单独点 Stage 2–4 | 提示需先完整流程或 Stage 1 |
| 运行中切换模型 | **自动停止**当前 API，需重新点击 Stage |

侧边栏可关闭「LLM 结果缓存」后再生成，避免同题命中旧模型缓存。

## 6. 历史模式

- **查看**：只读浏览备课包，可导出 Word；侧边栏索引可跳转 Stage。
- **载入**：写入新建页，可续跑未完成阶段或修改后重跑。
- **搜索 / 分页 / 收藏**：见历史列表；「只看收藏」按数据库筛选，分页与总数准确。

仅 **载入** 会同步该条的模型信息到新建页；**仅查看** 不会改变新建页当前的生成状态。

## 7. 导出与分享

- **一键导出 Word**：至少完成 Stage 1 后可用；含题目与已完成阶段，标题/列表/表格已排版。
- **下载 JSON**：含 `stage1_json` 与各阶段原文，便于二次加工。
- **分享链接**（需 Neon + `APP_BASE_URL`）：见 [DEPLOY_CLOUD.md](DEPLOY_CLOUD.md)。

## 8. 断点续跑

- 未完成四阶段时，新建页会出现 **续跑** 按钮（模型一致时）。
- 某 Stage 失败后可用 **重试 Stage N** 或单独 **重跑** 某一阶段。
- 换题时会自动清空本页缓存（**历史记录保留**）。

## 9. 常见问题

- **卡在 Calling API**：看运行状态字数是否增长；可换更快模型或增大 `.env` 超时，见 README「排错」。
- **范文词数不对**：Stage 2 要求 105–125 词；可单独重跑 Stage 2 或换指令跟随更好的模型。
- **Cloud 未更新**：`git push` 后在 Streamlit Cloud **Reboot**；核对侧边栏底部版本标签。
