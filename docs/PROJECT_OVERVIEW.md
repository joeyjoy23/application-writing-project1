# 高考英语应用文 AI 分析系统 — 项目简介

## 这是什么

面向高考英语**应用文**的 AI **备课助手**：固定四阶段工作流，不是自由聊天。教师粘贴一道真题，自动生成从审题到教案的成套材料，可导出 Word、自动保存历史、断点续跑。

适用：个人备课、教研讨论、课堂示范与讲评整理。

## 项目亮点

- **教研级流水线**：Stage1 输出结构化 JSON 写作蓝图，后续阶段严格据此生成，要点覆盖可校验，避免聊天式「每次不同、易漏要点」。
- **题目类型智能识别**：Stage1 自动识别题目类型（活动记录类 / 观点理由类 / 混合类型），并根据类型给出差异化的评价重点、高分要点和构思维度推荐，让分析更精准。
- **范文有层次、有风格**：PEEL 便签 + 基础版 + 情感共鸣型 / 逻辑思辨型两份高分范文 + 三版对比与升级解析；范文统一 **105–125 英文词**（忽略原题「约 80 词」等过短要求）；根据题目类型差异化生成范文风格。
- **语言与教案一体**：Stage3 功能句型 + 三层话题词汇（必备 / 进阶 / 亮点），根据题目类型优先推荐不同类型的句型和词汇；Stage4 按所选 **学生水平** 生成差异化教学路径、易错预警与课后练习。
- **为真实课堂设计**：Prompt 独立在 `prompts/`；多模型支持；Stage2/3 并行；可选 LLM 结果缓存；界面自动整理构思维度等 Markdown 排版。
- **可留存、可续用**：**每完成一个 Stage 自动写入历史**（同题同模型合并一条）；失败或停止仍保留已完成部分；一键 Word；云端 Neon 持久化；**Token 用量自动记录**，便于回溯成本。
- **API 容错机制**：调用失败自动重试（最多 3 次，指数退避），超时/连接失败/限流自动重试，配置错误不重试。

## 四阶段一览

**Stage1** 审题（JSON + 教师总结）→ **Stage2** PEEL 与多版范文 → **Stage3** 句型与词汇 → **Stage4** 学情适配教学指南。

可跑全流程，也可单跑某一阶段；侧边栏 **已生成内容索引** 可快速跳转（新建页与历史详情均可用）。

## 历史与清空（重要）

- **自动保存**：无需手动点保存；至少完成 Stage 1 后即写入历史库。
- **清空当前结果**：只清除当前页面的四阶段输出，**历史里的记录不会删除**。
- **删除**：仅在历史列表中操作，才会永久删掉某条备课包。
- **载入历史**：可续跑未完成阶段，或在新建页基于存档修改、重跑。

详细说明见 [USAGE.md](USAGE.md)。

## 换模型

更换侧边栏模型后，系统会识别当前结果与所选模型是否一致：

- **完整流程** → 用新模型重跑全部阶段  
- **Stage 1** → 清空后只重跑审题  
- 运行中换模型 → 自动停止当前请求  

## 快速使用

1. `streamlit run app.py`（或双击 `run.bat`）。
2. 侧边栏选模型、填 Key →「新建」粘贴题目 →「完整流程」或 Stage 1–4。
3. 导出 Word，或在「历史」搜索、查看、载入续跑。
